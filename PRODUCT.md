# Product

## Register

product

## Users

League of Legends players who want to compare midlane champion pools using patch-specific matchup data. They use the tool as a focused analysis workspace: select realistic candidates, run the optimizer, and inspect why a pool performs well.

## Product Purpose

Turn the existing champion-pool optimizer into an understandable interactive tool without changing its mathematical model. Success means candidate selection is fast, recommendations are reproducible, and matchup evidence is visible enough for users to judge small score differences.

## Brand Personality

Focused, analytical, game-aware. The interface should feel dense and capable without becoming noisy or theatrical.

## Anti-references

Avoid marketing-page composition, oversized cards, loose pill-heavy controls, decorative gradients, generic chart dashboards, and interfaces that hide the matchup data behind a single score.

## Design Principles

1. Put the selection and optimization workflow first.
2. Explain recommendations with the same matchup values used by the model.
3. Prefer compact, aligned lists and matrices over decorative cards.
4. Keep state changes immediate and predictable.
5. Use color to communicate matchup strength and status, not as decoration.

## Accessibility & Inclusion

Use readable contrast, visible keyboard focus, semantic controls, reduced-motion support, and text values alongside color encodings. The interface should remain usable without champion icons and on narrower screens.
